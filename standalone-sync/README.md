# CODESYS Git Sync Standalone Tool

Этот каталог представляет собой **автономный набор инструментов** для двусторонней синхронизации проектов **CODESYS V3 / Abak.IDE** с текстовыми файлами Structured Text (`.st`). 

Вы можете открыть эту папку как отдельный проект/рабочую область в новой сессии **VS Code** или ИИ-ассистента **Antigravity**. Инструмент работает автономно, **без привязки к MCP-серверу** и без необходимости запуска каких-либо дополнительных фоновых портов или инспекторов.

---

## 📦 Особенности и зависимости
*   **Zero-Dependency (Node.js)**: Скрипты синхронизации используют только стандартные библиотеки Node.js. Установка сторонних пакетов через `npm install` **не требуется**.
*   **Портативность**: Вы можете перенести эту папку в любое место на диске. Она содержит всё необходимое для запуска.
*   **Python**: Для графического интерфейса используется встроенная библиотека `tkinter`. Убедитесь, что Python установлен в системе.

---

## 🤖 Как работать через Antigravity (ИИ-ассистента)

Когда вы открываете эту папку в новой сессии Antigravity, ИИ-ассистент имеет доступ к инструментам командной строки (терминалу). Вы можете общаться с ним на естественном языке.

### Примеры команд для ИИ-ассистента:

1.  **Выгрузка кода проекта на диск (Экспорт)**:
    > *"Антигравити, сделай выгрузку кода проекта CODESYS"*
    или
    > *"Антигравити, выгрузи проект из D:\Projects\CEX_15\апп 511\PLC в папку project_sources"*
    ИИ самостоятельно выполнит команду `node sync_cli.js export ...` через терминал.

2.  **Загрузка измененного кода обратно (Импорт)**:
    > *"Антигравити, залей измененный код обратно в проект"*
    или
    > *"Антигравити, импортируй ST файлы из project_sources в проект CODESYS"*
    ИИ выполнит `node sync_cli.js import ...` для синхронизации ваших правок.

3.  **Создание нового кода**:
    Вы можете попросить ассистента:
    > *"Создай новый метод CalcAverage для функционального блока FB_ValveLogic"*
    Ассистент создаст файл `.method.st` в нужной подпапке и предложит вам запустить импорт в CODESYS.

## 🛠️ Способы ручного запуска (Без ИИ)

### 1. Встроенные задачи VS Code (Tasks)
Нажмите `Ctrl+Shift+P` -> выберите `Tasks: Run Task` (Запуск задачи) -> выберите одну из настроенных задач:
*   `CODESYS: Export sources (Выгрузить в .st)` — быстрая выгрузка проекта по умолчанию.
*   `CODESYS: Import sources (Залить код в CODESYS)` — быстрый импорт изменений по умолчанию.
*   `CODESYS: Export sources [Custom Paths]` — откроет поле ввода вверху окна VS Code, где вы сможете указать или подтвердить пути к проекту и папке выгрузки.
*   `CODESYS: Import sources [Custom Paths]` — интерактивно запросит пути для импорта изменений обратно.

### 2. Standalone GUI (Графический интерфейс)
Запустите графическое окно двойным кликом по файлу `manage.bat` или выполнив:
```bash
python manager.py
```
В открывшемся компактном окне:
*   Нажмите на кнопки с тремя точками `...` рядом с полями путей, чтобы выбрать `.project` файл проекта (или папку) и папку с `.st` исходниками.
*   Нажмите **"Экспорт исходников"** для выгрузки.
*   Нажмите **"Импорт исходников"** для применения ваших правок.
Прогресс выполнения отобразится в отдельном текстовом окне логов.

### 3. Прямой запуск CLI из консоли
Вы можете использовать терминал напрямую:
```bash
# Экспорт (выгрузка)
node sync_cli.js export "<путь_к_проекту_или_папке>" "<папка_для_st_файлов>"

# Импорт (загрузка обратно)
node sync_cli.js import "<путь_к_проекту_или_папке>" "<папка_для_st_файлов>"
```
*Если требуется переопределить путь к среде разработки или профиль, используйте флаги `--codesys-path` и `--codesys-profile`.*

## 📁 Структура папок при экспорте
При экспорте проект CODESYS раскладывается на иерархическую структуру папок, соответствующую дереву объектов среды разработки:
*   Программы: `*.prg.st`
*   Функциональные блоки: `*.fb.st`
*   Функции: `*.func.st`
*   Методы блоков: `*.method.st`
*   Свойства блоков: `*.property.st`
*   Действия: `*.action.st`
*   Глобальные переменные (GVL): `*.gvl.st`
*   Типы данных (DUT): `*.dut.st`

Каждый текстовый файл содержит заголовки `// @OBJECT_ID: <guid>` для прочной связи с базой данных проекта CODESYS, а также разделы объявления переменных (`// @DECLARATION`) и тела кода (`// @IMPLEMENTATION`), что делает правку кода в VS Code простой и естественной.
